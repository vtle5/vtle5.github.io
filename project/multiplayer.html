TCP, UDP?

For the longest of time, I've always wanted to develop a multiplayer game. It was the summer of 2020, and with nothing else to do I felt it was time to explore creating one. While many tutorials offered easy methods to network a game, I wanted something that didn't rely on an existing service. I eventually found Tom Weiland. He gave a tutorial on networking a game using TCP and UDP, providing insight on how these protocols worked. The framework he provided functioned very well, and now I had an amazing base to work off of. I began recoding the framework so it would be compatible with what I needed and turned it into a 2D engine. When working with networking, it requires a very different way of thinking when coding in functionality. It took time and there were plenty of issues with the prototype.

Network Issues

Initial testing was successful, the prototype had functioned and I was able to play with friends across the country! Of course there were issues. A major one being that the localhost must be the last one to connect to the server. Another issue is that the game would occasionally lag horribly with bullets being the greatest offender. On occasion, bullets would freeze and refuse to despawn. Fortunately, the players would never de-sync and the game was stable for the most part. These issues would be adressed in later versions of the project that are not open to the public currently.

Client Prediction and Updates

To deal with the issue of player joining, a lobby system has been added so that players would sync once the game starts. It was later found that bullets were using a lot of packets which would degrade network quality. Client prediction in game design is a major issue and extremely complex solutions have been made to minimize these issues. Using what I learned from research, I decided to add a time stamp to bullet packets. This would allow clients to handle most bullet calculations and have them appear the same for all players with use of the time stamp. With the continued development of the game, I decided to create a game launcher using XAML and .NET. This allows users to easily update their clients without needing to manually download the project again.
